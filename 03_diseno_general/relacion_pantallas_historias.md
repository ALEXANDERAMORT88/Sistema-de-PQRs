# Relación Panatalla -> Historia 

Porque: 
- Las historias dicen QUÉ necesita el usuario.
- Las pantallas muestran CÓMO lo va hacer. 

La ide es que si una pantalla no responde a una historia, no deberia de existir. 

| Pantalla                |                         Justificación                                   |  Historia de Usuario |
|-------------------------|-------------------------------------------------------------------------|----------------------|
| Registro de Usuario     | Permite crear la cuenta, Sin esta pantalla no existe acceso al sistema  |  HU-01               |
| Inicio de Sesión        | Permite ingresar a la cuenta                                            |  HU-01               |
| Panel Principal         | Permite el acceso a registrar y consultar PQR                           |  HU-02 / HU-03       |
| Registrar PQR           | Captura tipo, descripción y prioridad, brinda la creación del radicado  |  HU-02               |
| Confirmación de PQR     | Es parte del flujo feliz de la HU                                       |  HU-02               |
| Consulta de PQR         | Consultar estado, fechas y tiempos                                      |  HU-03               |
